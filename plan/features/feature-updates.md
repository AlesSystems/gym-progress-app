# Feature Updates

This document outlines planned feature changes and additions for the gym progress app.

---

## 1. Remove PR Feature in Workout

**Change:** Remove the Personal Record (PR) feature from the workout experience.

**Scope:** All PR-related UI, logic, and calculations within the workout flow will be removed. Users will no longer see PR indicators or PR-based feedback during active workout sessions.

---

## 2. Add Cancel Workout Feature

**Change:** Allow users to cancel an in-progress workout.

**Scope:** Users should be able to abandon or cancel a workout session that is currently in progress. The session should be marked appropriately (e.g., as cancelled or abandoned) and not counted as a completed workout. This provides flexibility when plans change mid-session.

---

## 3. Add New Exercise During Workout (Ad-hoc Exercises)

**Change:** Allow users to add a new exercise during a workout that does not exist in the main exercises page/library.

**Scope:** Users can create and log exercises on-the-fly while in an active workout session, without having to pre-add them to the exercise library. These ad-hoc exercises should be usable for that session and optionally saved for future use.

---

## 4. Add Button to Delete Old Sessions from History

**Change:** Provide a way for users to delete past workout sessions from their history.

**Scope:** A delete button (or similar control) should be available in the history view so users can remove old sessions they no longer want to keep. This supports data hygiene and user control over their records.

---

## 5. Remove RPE Feature from the App

**Change:** Remove the Rate of Perceived Exertion (RPE) feature from the application.

**Scope:** All RPE-related inputs, displays, and data storage will be removed. Users will no longer log or view RPE values for sets or exercises.

---
